Permission Changes

Starting Mar 9, 2019 Google bans apps with CallLog and SendSMS permissions. They claim these to be dangerous permissions and only allow certain types of apps to get an exclussion to obtain these permissions. The CallLog permissions is what's needed to identify incoming calls. While the SendSMS permission is required to be able to send SMS, or auto-respond.

We have tried to obtain an exceptions for Silence, claiming it to be a legitimate use case for these permissions. Yet each time Google has responded with a denial, without clarifying the reason. In light of this, any update, whether bug fix or a feature update, must comply with this new policy. And the latest version of the app has the following features removed to comply with this policy:

  • Call identification via group. This is now restricted to Favorites only.
  • Flexible repeated call options can't be supported without caller id. Basic repeated call feature is supported via standard DND api.
  • Sending of SMS in response to incoming calls or SMS isn't possible anymore.

We are looking at alternatives for implementing the listed features. Possible choices can include a companion app published via non Google store. We will make every attempt to get removed functionality re-integrated, if practically possible. Thank you for your support and understanding.
